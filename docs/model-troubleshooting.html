model-troubleshooting

Introduction

This page is for documenting the model checking procedures for the maximum size estimation models. The models were all fit originally using the hierarchical negative binomial model code/stan/hierarchical_nb_max_size_stable. With these fits, we identified a number of issues, so we are now working backwards to simpler models. I outline below the steps to keep a record for future.

Divergent transitions

The original models had ~700, or about 50%, that exhibited divergent transitions after warmup. I should have realized at the outset that we are likely to have many taxa with only a single site, so the hierarchical structure of the model is obviously not appropriate. It turned out that ~400 only have a single site.

get_divergence_draws <- function(
    fit,
    variables
) {
  draws <- as.data.frame(
    fit$draws(
      variables = variables,
      format = "df"
    )
  )

  diagnostics <- as.data.frame(
    fit$sampler_diagnostics(
      format = "df"
    )
  )

  draw_key <- paste(
    draws$.chain,
    draws$.iteration,
    sep = ":"
  )

  diagnostic_key <- paste(
    diagnostics$.chain,
    diagnostics$.iteration,
    sep = ":"
  )

  diagnostic_row <- match(
    draw_key,
    diagnostic_key
  )

  if (anyNA(diagnostic_row)) {
    stop(
      "Could not match all posterior draws ",
      "to sampler diagnostics."
    )
  }

  draws$divergent__ <-
    diagnostics$divergent__[diagnostic_row]

  draws$energy__ <-
    diagnostics$energy__[diagnostic_row]

  draws$accept_stat__ <-
    diagnostics$accept_stat__[diagnostic_row]

  draws$treedepth__ <-
    diagnostics$treedepth__[diagnostic_row]

  draws
}

screen_divergent_parameters <- function(
    fit,
    variables
) {
  dat <- get_divergence_draws(
    fit = fit,
    variables = variables
  )

  is_divergent <- dat$divergent__ == 1

  if (!any(is_divergent)) {
    return(
      data.frame(
        variable = variables,
        n_divergent = 0L,
        ks_distance = NA_real_,
        median_shift_mad = NA_real_,
        divergent_rank_median = NA_real_,
        divergent_rank_iqr = NA_real_
      )
    )
  }

  results <- lapply(
    variables,
    function(variable) {
      x <- dat[[variable]]

      keep <- is.finite(x)

      x <- x[keep]
      div <- is_divergent[keep]

      x_div <- x[div]
      x_ok <- x[!div]

      if (
        length(x_div) < 2L ||
        length(x_ok) < 2L
      ) {
        return(
          data.frame(
            variable = variable,
            n_divergent = length(x_div),
            ks_distance = NA_real_,
            median_shift_mad = NA_real_,
            divergent_rank_median = NA_real_,
            divergent_rank_iqr = NA_real_
          )
        )
      }

      parameter_scale <- stats::mad(
        x,
        constant = 1.4826
      )

      if (
        !is.finite(parameter_scale) ||
        parameter_scale <= 0
      ) {
        parameter_scale <- stats::sd(x)
      }

      if (
        !is.finite(parameter_scale) ||
        parameter_scale <= 0
      ) {
        parameter_scale <- 1
      }

      ks_result <- suppressWarnings(
        stats::ks.test(
          x_div,
          x_ok,
          exact = FALSE
        )
      )

      scaled_ranks <-
        rank(
          x,
          ties.method = "average"
        ) / (length(x) + 1)

      div_ranks <- scaled_ranks[div]

      data.frame(
        variable = variable,
        n_divergent = length(x_div),

        median_divergent =
          stats::median(x_div),

        median_nondivergent =
          stats::median(x_ok),

        ks_distance =
          unname(ks_result$statistic),

        median_shift_mad =
          abs(
            stats::median(x_div) -
              stats::median(x_ok)
          ) / parameter_scale,

        divergent_rank_median =
          stats::median(div_ranks),

        divergent_rank_iqr =
          stats::IQR(div_ranks)
      )
    }
  )

  results <- do.call(
    rbind,
    results
  )

  results[
    order(
      results$ks_distance,
      decreasing = TRUE,
      na.last = TRUE
    ),
  ]
}

make_bayesplot_nuts <- function(fit) {
  diagnostics <- as.data.frame(
    fit$sampler_diagnostics(
      format = "df"
    )
  )

  diagnostic_variables <- setdiff(
    names(diagnostics),
    c(
      ".chain",
      ".iteration",
      ".draw"
    )
  )

  long_list <- lapply(
    diagnostic_variables,
    function(variable) {
      data.frame(
        Chain = diagnostics$.chain,
        Iteration = diagnostics$.iteration,
        Parameter = variable,
        Value = diagnostics[[variable]]
      )
    }
  )

  do.call(
    rbind,
    long_list
  )
}

Check posteriors for some taxa with initializations near boundary

Some of the initial values for the tau_log_lambda parameters run up against the set boundaries for this parameters. The value is set at an upper range of 4. This parameter characterizes the across site expectation in log expected counts. We bounded this to avoid drawing prior draws that lead to Inf values during sampling. However, if counts vary greatly among sites, our initial values might exceed this limit leading to rejection of some or all chains. For context, this parameter represents a multiplicative difference in counts, therefore with a limit of 4, as set, corresponds to a 55x difference in counts for a site that is one-standard deviation. I include a step to check the initialization values for better model specification in the future.
